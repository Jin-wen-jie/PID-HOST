# PID 自动整定设计规格

日期：2026-07-09

## 目标

在现有 PID-HOST 上位机基础上增加标准自动整定功能。用户选择电机通道并设置安全边界后，上位机自动执行一次或多轮阶跃测试，采集 `SP/PV/OUT` 遥测，计算响应指标，迭代下发 `Kp/Ki/Kd`，最终给出并保留表现最好的 PID 参数。

自动整定只负责“测试、分析、下发参数”。PID 实时闭环仍由下位机执行，上位机不接管实时控制输出。

## 已确认范围

- 需求档位：标准实现版。
- 适用对象：现有两路电机 PID，`ch=0` 为电机1，`ch=1` 为电机2。
- 通信协议：复用现有 JSON Lines 协议，不新增下位机必需命令。
- 控制边界：允许上位机在安全上限内自动下发多组 `Kp/Ki/Kd`。
- 测试方式：允许自动改变一次 `SP` 做阶跃测试。
- 数据来源：使用下位机周期上报的 `tel` 遥测帧，包括 `sp/pv/out`。
- 运行模式：真实串口模式和模拟数据模式都要能跑通基础流程。

## 非目标

- 不把 Windows 上位机放进实时闭环。
- 不直接输出 PWM、电压、电流等实时控制量。
- 不要求下位机实现新的自动整定算法。
- 不实现复杂模型辨识、频域分析或多算法插件系统。
- 不同时整定两路电机；每次只整定一个选中通道。
- 不自动突破用户设置的 PID、SP、PV、OUT 安全边界。

## 用户流程

1. 用户选择电机通道。
2. 用户设置整定目标 `SP`、PID 上下限、最大轮数和每轮采集时长。
3. 用户点击“开始整定”。
4. 软件保存当前 PID 和 SP 作为回滚值。
5. 软件按当前通道发送测试 PID 和测试 SP。
6. 软件采集该通道一段 `tel` 数据。
7. 软件计算响应指标并给本轮评分。
8. 软件生成下一轮 PID，继续测试。
9. 达标、达到最大轮数、用户停止或触发安全条件时结束。
10. 结束后显示最优参数，用户可保留最优参数或回滚到整定前参数。

## 界面设计

在现有左侧参数面板下方新增“自动整定”区域。

控件：

- 启用整定目标 `SP` 输入。
- `Kp/Ki/Kd` 最小值和最大值输入。
- 最大轮数输入，默认 6。
- 每轮采集时长输入，默认 4 秒。
- 最大允许超调输入，默认 30%。
- 最大允许输出饱和时长输入，默认 1 秒。
- “开始整定”按钮。
- “停止整定”按钮。
- “保留最优参数”按钮。
- “回滚整定前参数”按钮。

状态显示：

- 当前状态：空闲、准备、采集中、评估中、已达标、已停止、已失败。
- 当前轮次。
- 当前测试参数。
- 最优参数。
- 响应指标：超调、稳态误差、调节时间、振荡分数、输出饱和比例。
- 停止原因或失败原因。

交互约束：

- 整定运行中禁用手动发送 PID 和手动发送目标值，避免命令互相覆盖。
- 整定运行中允许用户停止整定。
- 串口断开、命令超时或协议错误时自动停止整定。
- 模拟模式下可以启动整定，用于无设备验证流程。

## 核心模块

### `auto_tune.py`

新增自动整定核心模块，不依赖 PySide6。

职责：

- 定义整定配置 `AutoTuneConfig`。
- 定义每轮结果 `AutoTuneTrial`。
- 定义整定状态 `AutoTuneSession`。
- 从遥测数据计算响应指标。
- 判断是否达标。
- 判断安全停止条件。
- 根据上一轮结果生成下一轮 PID。

核心输入：

- 当前通道。
- 初始 PID 和 SP。
- PID 上下限。
- 采集到的 `TelemetrySample` 列表。

核心输出：

- 下一轮 PID。
- 本轮响应指标。
- 当前最优 PID。
- 整定状态。
- 停止原因。

### `MainWindow`

扩展现有主窗口，负责编排 UI、命令发送和遥测喂入。

职责：

- 创建自动整定控件。
- 在开始整定时读取用户配置。
- 保存整定前参数。
- 调用 `AutoTuneSession` 获取下一轮 PID。
- 复用 `_send_command("set_pid")` 和 `_send_command("set_sp")` 下发命令。
- 在 `add_telemetry()` 中把选中通道样本喂给整定会话。
- 在每轮采集结束后刷新 UI 状态。
- 处理停止、失败、保留、回滚。

## 整定策略

第一版使用保守阶跃整定策略，优先保证安全和可预测性。

初始参数：

- 第一轮从当前界面 PID 开始。
- 如果当前 PID 超出用户设置边界，开始前提示并拒绝启动。

评分指标：

- 稳态误差越小越好。
- 超调越小越好。
- 调节时间越短越好。
- 振荡越小越好。
- 输出长时间饱和扣分。

达标条件：

- 稳态误差低于目标阈值。
- 超调低于用户设置上限。
- 最近一段 PV 波动低于振荡阈值。
- OUT 没有持续饱和。

参数调整规则：

- 如果响应太慢且无明显超调，适度增加 `Kp`。
- 如果存在稳定偏差，适度增加 `Ki`。
- 如果超调或振荡明显，降低 `Kp` 或增加 `Kd`。
- 如果输出长时间饱和，降低 `Kp/Ki` 并停止激进尝试。
- 每轮调整幅度随轮次递减。
- 所有新参数必须夹在用户设置的上下限内。

第一版不追求理论最优，只追求比人工盲调更稳定、更可解释，并且可以被测试覆盖。

## 数据流

1. 用户点击“开始整定”。
2. UI 创建 `AutoTuneSession`。
3. Session 给出第一轮 PID。
4. UI 发送 `set_pid`。
5. UI 发送 `set_sp`。
6. 下位机继续上报 `tel`。
7. UI 把当前通道 `tel` 样本传给 Session。
8. 采集时间到后，Session 计算指标和评分。
9. Session 判断达标、失败或继续。
10. 继续时生成下一轮 PID，回到步骤 4。
11. 结束时 UI 显示最优参数和停止原因。

## 安全策略

启动前校验：

- 必须选择有效通道。
- 必须处于串口连接或模拟模式。
- PID 上下限必须有效，最小值不能大于最大值。
- 当前 PID 必须在边界内。
- 目标 SP 必须是有限数字。

运行中保护：

- 每轮采集有最大时长。
- 最大轮数到达后自动停止。
- 命令超时自动停止。
- 串口断开自动停止。
- PV 长时间没有变化或没有新遥测自动停止。
- 超调超过安全上限自动停止。
- OUT 持续饱和超过阈值自动停止。
- 用户点击“停止整定”立即停止下一轮动作。

结束处理：

- 正常达标：保留最优参数到界面，但不自动保存配置文件。
- 达到最大轮数：保留当前最优参数，提示未完全达标。
- 安全停止或失败：停止下发新参数，提示原因，允许一键回滚。
- 用户停止：停止下发新参数，保留当前界面参数，允许一键回滚。

## 异常处理

- `set_pid` 返回 `err`：停止整定，显示下位机错误。
- `set_sp` 返回 `err`：停止整定，显示下位机错误。
- `tel` 字段非法：沿用现有协议层丢弃逻辑，不进入整定样本。
- 当前通道切换：整定运行中禁止切换通道。
- 配置加载：整定运行中禁止加载参数配置。
- 录制 CSV：可与整定同时运行，便于复盘。

## 测试设计

单元测试：

- 响应指标计算：超调、稳态误差、调节时间、振荡分数、输出饱和比例。
- 达标判断：给定稳定响应时返回达标。
- 安全停止：超调过大、输出持续饱和、无新遥测时停止。
- 参数生成：下一轮 PID 不越界，且慢响应、振荡、稳态误差场景分别走对应调整方向。
- 会话状态机：开始、采集、评估、继续、达标、失败、用户停止。

UI 测试：

- 自动整定控件存在。
- 开始整定后禁用手动发送 PID 和 SP。
- 停止整定后恢复手动按钮。
- 整定按当前选中通道发送 `set_pid` 和 `set_sp`。
- 模拟模式下可以完成至少一轮采集和评估。

回归测试：

- 现有协议、数据缓存、模拟器、启动器测试保持通过。
- 原有手动调参流程不受影响。

## 验收标准

- 用户可以在模拟模式下启动自动整定，完成至少一轮测试并看到指标。
- 用户可以在真实串口模式下对选中电机通道启动整定。
- 整定运行中可以停止，并可回滚到整定前参数。
- 所有自动下发的 PID 都在用户设置边界内。
- 出现串口断开、命令超时、超调过大或输出持续饱和时自动停止。
- 现有测试全部通过，新增自动整定测试覆盖核心状态机和指标计算。

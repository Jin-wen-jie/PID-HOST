# PID-HOST 软件设计规格

版本：v0.1
日期：2026-07-09

## 目标

PID-HOST 是一个 Windows 桌面 PID 上位机，用于通过 UART/USB 转串口连接下位机，完成单路 PID 参数设置、实时曲线监控、事件日志和 CSV 数据记录。

第一版定位为“调参 + 监控 + 记录”工具。PID 闭环计算在下位机运行，上位机不承担实时控制输出，避免串口延迟、系统调度和电脑断连影响闭环稳定性。

## 已确认范围

- 需求档位：标准实现版。
- 技术栈：Python + PySide6 + PyQtGraph。
- 通信方式：串口 UART/USB 转串口。
- 协议：JSON Lines，每条消息是一行 UTF-8 JSON，以换行结束。
- PID 通道：第一版单路 PID，通道号固定为 `0`，协议预留 `ch` 字段。
- 数据频率：遥测上报 10-20 Hz，默认 20 Hz。
- 实时曲线：显示 `SP`、`PV`、`OUT` 三条曲线。
- PID 计算位置：下位机运行 PID，上位机设置参数并显示遥测。
- 主布局：曲线主导布局。
- 测试辅助：包含模拟数据模式，便于无下位机时验证界面、曲线和录制。

## 非目标

第一版不实现多通道 PID、自动整定、上位机实时 PID 控制、二进制帧协议、复杂插件系统、在线协议编辑器和高级控制性能分析。这些功能可在第一版稳定后扩展。

## 总体架构

软件采用分层标准结构：

- `ui`：PySide6 界面、控件状态、用户交互、PyQtGraph 曲线显示。
- `app`：业务编排，包括连接状态、命令序号、参数校验、命令超时、数据缓存、录制状态和日志事件。
- `protocol`：JSON Lines 编码、解码、消息校验和 `ack/err` 匹配。
- `serial`：串口枚举、打开/关闭、读写 worker、收发队列和异常上报。
- `data`：遥测缓存、曲线窗口数据、CSV 写入。
- `config`：参数配置保存/加载，例如串口设置、PID 参数和曲线显示设置。

层间依赖方向：

```text
ui -> app -> protocol
         -> serial
         -> data
         -> config
```

`serial` 层不直接依赖 UI。串口读到的数据通过信号、队列或回调交给 `app`，再由 UI 主线程刷新界面。

## 数据流

1. 用户在 UI 输入 `Kp/Ki/Kd` 或 `SP`。
2. `app` 校验参数并生成带 `seq` 的命令。
3. `protocol` 编码为 JSON 行。
4. `serial` 通过串口发送到下位机。
5. 下位机回 `ack` 或 `err`。
6. 下位机周期上报 `tel` 遥测。
7. `protocol` 解码并校验遥测字段。
8. `data` 更新缓存和 CSV。
9. `ui` 定时刷新实时曲线和数值显示。

## 串口物理参数

默认串口参数：

- 波特率：`115200`
- 数据位：`8`
- 校验位：`None`
- 停止位：`1`
- 流控：无硬件流控、无软件流控
- 编码：UTF-8
- 行结束：发送统一使用 `\n`，接收兼容 `\n` 和 `\r\n`

UI 允许用户修改波特率；数据位、校验位、停止位和流控第一版固定为默认值。

## 协议概要

每条消息是一行 JSON 文本。发送方必须以 `\n` 结束，接收方按行切分；解析前去除行尾 `\r` 和 `\n`。

上位机发送：

```json
{"type":"hello","seq":1}
{"type":"set_pid","seq":2,"ch":0,"kp":1.2,"ki":0.05,"kd":0.01}
{"type":"set_sp","seq":3,"ch":0,"sp":50.0}
{"type":"stream","seq":4,"enabled":true,"rate_hz":20}
```

下位机回复：

```json
{"type":"hello","seq":1,"device":"pid-mcu","fw":"1.0.0","proto":1}
{"type":"ack","seq":2}
{"type":"err","seq":3,"code":"bad_value","message":"sp out of range"}
{"type":"tel","ch":0,"t":123456,"sp":50.0,"pv":48.7,"out":32.1}
```

完整协议见 [下位机协议手册](protocol-for-mcu.md)。

## 命令模型

第一版采用简单可靠的逐一应答模型：

- 同一时间只允许一个待确认命令。
- 上位机发送命令后等待相同 `seq` 的 `ack` 或 `err`。
- 等待期间禁用同类发送按钮，避免命令交错。
- `ack_timeout_ms` 默认 `1000`。
- 超时后该命令标记失败，记录日志，允许用户重新发送。
- `tel` 遥测不受命令等待影响，可以持续接收。

该模型吞吐不高，但足够覆盖 10-20 Hz 遥测和低频调参命令，也便于 MCU 端实现。

## 参数校验

上位机本地校验：

- `Kp/Ki/Kd/SP` 必须是有限数字，不能是空值、NaN 或 Infinity。
- 第一版默认不写死业务上下限，避免误伤不同设备；上下限后续可放入配置文件。
- `rate_hz` 限制为 `10` 到 `20`。
- `ch` 第一版必须为 `0`。

下位机仍需重复校验所有参数，并在非法时返回 `err`。

## 错误码

第一版错误码：

| code | 含义 |
| --- | --- |
| `bad_json` | JSON 解析失败 |
| `bad_type` | 未知或不支持的消息类型 |
| `missing_field` | 缺少必填字段 |
| `bad_value` | 字段值非法或超出下位机允许范围 |
| `unsupported` | 功能或协议版本不支持 |
| `busy` | 下位机暂时不能处理该命令 |

上位机收到 `err` 后必须记录 `code/message`，并把对应操作标记为失败。

## 界面设计

第一版使用曲线主导布局。

顶部工具栏：

- 串口选择。
- 刷新串口。
- 波特率选择，默认 115200。
- 连接/断开按钮。
- 设备握手状态。
- 固件版本和协议版本显示。
- 开始/停止 CSV 录制。
- 保存/加载参数配置。

左侧参数面板：

- `Kp`、`Ki`、`Kd` 输入。
- `SP` 目标值输入。
- 发送 PID 参数按钮。
- 发送目标值按钮。
- 当前 `SP/PV/OUT` 数值显示。
- 参数输入必须通过本地校验；非法输入不发送串口。

主区域：

- PyQtGraph 实时曲线。
- 三条曲线：`SP`、`PV`、`OUT`。
- 支持暂停/继续。
- 支持清空曲线。
- 默认显示最近 60 秒数据。
- 曲线数据来自有效 `tel` 帧。

底部日志区：

- 显示连接、断开、发送命令、收到 `ack`、收到 `err`、解析失败、串口异常。
- 支持可选显示原始收发帧。

## 线程模型

- UI 主线程：创建窗口、响应用户操作、刷新曲线和控件状态。
- 串口 worker：执行阻塞式串口读写，逐行读取 JSON Lines。
- UI 刷新定时器：按固定周期从数据缓存读取最近窗口数据并刷新曲线。

线程安全约束：

- 串口 worker 不直接操作 UI 控件。
- 跨线程数据通过 Qt signal/slot、线程安全队列或受控回调传递。
- 遥测缓存由 `app/data` 层集中更新；如果存在跨线程写读，使用锁或主线程归并。
- CSV 写入由 `data` 层串行处理，避免多线程同时写同一文件。

## 数据记录

CSV 录制由用户手动开始和停止。

文件名格式：

```text
pid_log_YYYYMMDD_HHMMSS.csv
```

字段：

```text
pc_time,device_time_ms,ch,sp,pv,out
```

行为：

- 开始录制后，每条有效 `tel` 写入一行。
- 停止录制时关闭文件。
- 串口断开时自动停止录制并关闭文件。
- JSON 解析失败或字段缺失的遥测不写入 CSV。

## 异常处理

- 串口打不开：提示端口不存在、权限不足或被占用，不进入连接状态。
- 串口断开：停止发送命令，停止录制，日志提示断开。
- JSON 解析失败：丢弃该行，日志记录原始内容。
- 遥测字段缺失：丢弃该帧，避免污染曲线和 CSV。
- 命令超时：发送后未在 `ack_timeout_ms` 内收到相同 `seq` 的 `ack/err`，日志提示超时。
- 下位机返回 `err`：日志显示 `code` 和 `message`，对应操作不标记为成功。
- 输入非法参数：UI 本地拦截，不发串口。

## 测试设计

单元测试：

- `protocol`：编码、解码、坏 JSON、缺字段、未知类型、`ack/err` 匹配。
- `data`：曲线缓存长度、最近 60 秒窗口、CSV 字段和写入。
- `app`：参数校验、命令序号递增、命令超时、连接状态切换。

集成测试：

- 使用模拟串口或模拟数据源发送 `tel/ack/err`。
- 验证 UI 能在无下位机时通过模拟数据模式显示曲线。
- 验证录制 CSV 后文件字段和行数正确。

手动验证：

- 无设备：启动模拟数据模式，检查曲线、暂停、清空、录制。
- 有设备：连接串口，发送 `hello`，确认设备信息显示。
- 调参流程：发送 PID 参数，收到 `ack`，曲线继续更新。
- 错误流程：下位机返回 `err`，界面日志显示错误。
